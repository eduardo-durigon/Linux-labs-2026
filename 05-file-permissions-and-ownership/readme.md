🧪 Lab 05 – Linux File Permissions & Ownership

🎯 Summary
In this lab, I explored how Linux manages file permissions and ownership, which are fundamental concepts for system security and access control.
I practiced creating files and directories, inspecting their permissions, and modifying access using chmod. I also changed file ownership with chown and tested how permissions affect different users in real scenarios.
A key part of the lab was understanding how read, write, and execute permissions are assigned to the owner, group, and others, both in symbolic and numeric formats.

🔑 Key Takeaways
Linux permissions are divided into owner, group, and others
chmod modifies permissions (symbolic and numeric modes)
chown changes file ownership and directly impacts control over files
Only the owner or root can modify permissions
Permission errors like “Operation not permitted” are part of normal system security
The principle of least privilege can be applied by restricting access (e.g., chmod 700)

🧠 Reflection
This lab helped me understand how access control works in Linux at a practical level.
I learned that permissions are not just commands to memorize, but a core part of system security that determines who can read, modify, or execute files.
Troubleshooting issues like permission denial and ownership conflicts gave me a better understanding of how real systems enforce security boundaries.
